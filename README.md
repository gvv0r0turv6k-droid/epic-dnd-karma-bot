# Epic DND Karma Bot

Discord-бот для кармических бросков в стиле D&D. Бот рассчитан на один сервер, хранит данные в локальном SQLite-файле и не требует MySQL, Python, Visual Studio Build Tools или `node-gyp`.

## Что умеет бот

- ГМ создаёт запрос броска для конкретного игрока через `/gm-roll-request`.
- Под запросом броска появляется кнопка **🎲 Бросить** для нужного игрока.
- Игрок выполняет запрос кнопкой или простой командой `/roll` в том же канале.
- Результат публикуется красивым embed-сообщением и пингует ГМа.
- Если у игрока есть рероллы, под результатом появляется кнопка **🔁 Реролл: N доступно**.
- При нажатии кнопки реролла старая кнопка с предыдущего сообщения удаляется.
- Если после реролла у игрока ещё остались рероллы, под новым результатом появляется новая кнопка реролла.
- Карма скрыта от обычных игроков.
- Карма влияет на будущие броски: после провалов игроку становится легче получить успех, после успехов — сложнее.
- ГМ получает GM-аудит броска embed-сообщением: точное изменение кармы, скрытый исход и ссылку на сообщение с результатом.
- ГМ может выдавать, забирать и устанавливать рероллы через `/gm-rerolls`.
- Игрок получает красивое embed-уведомление, когда ГМ меняет его рероллы.
- Все основные успешные ответы команд оформлены embed-сообщениями.
- Есть обычные кубы без кармы через `/dice`.

## Требования

Нужен Node.js `22.13.0+`. Рекомендуется Node.js 24 LTS или новее.

Бот использует встроенный модуль Node.js `node:sqlite`, поэтому отдельная SQLite-библиотека и Python не нужны.

## Команды

```text
/gm-roll-request  — ГМ создаёт запрос броска для игрока
/roll             — игрок выполняет активный запрос броска
/reroll           — игрок тратит 1 реролл на повтор последней проверки
/dice             — обычные кубы без кармы
/profile          — профиль игрока
/gm-rerolls       — выдать/забрать/установить рероллы
/gm-karma         — изменить карму игрока
```
